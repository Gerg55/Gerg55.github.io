C Engine (in development)

Current features:

  • 3D first person camera control
  • .ply format model loading
  • A simple Blinn-Phong lighting model
  • Real-time directional shadow mapping
  • Console commands to modify scene (will be replaced with a GUI)
  • 3D transformation matrices (quaternions for rotation)
  • Simple material system

Recent additions (28/3/25):

  • Bad AABB collisions + physics (moving to ECS)
  • Font atlas text rendering (won't be touching .ttf for a while)

Work in progress (28/3/25):

  • GUI system -- currently working on
  • Began work on a simple ECS system
  • Skeletal animations (stopped until I figure out a way to export and load armatures in a way I like)

About:

This is not a proper game engine yet. Instead it is more a framework to easily create 3D graphics. Soon I will add simple phyics to change it from a 3D graphics framework to a more complete game framework. After that I will start implementing more game enginy features such as an editor and scripting(maybe) as well as further improve the graphical side. It is still early in the project, but once I have a decent game framework I might make a small game. I am not working on the project as much as I want myself to so I don't want to be too ambitious. Below is a video of the game engine as of 17/3/25. It demonstrates the transformations (translation, scaling and rotation) and the new material system which includes texturing and specular mapping. The specular mapping is not apparent in the video because I had forgot to transform the fragment position in the vertex shader so the specular highlights were wrong.
