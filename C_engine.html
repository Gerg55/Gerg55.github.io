C Engine

Current features:

  • .gply format model loading (includes skeletons and animations)
  • Standard shader using a Blinn-Phong lighting model
  • Simple material system
  • Debug graphics
  • Font atlas text rendering (won't be touching .ttf for a while)

Recent additions / changes (15/06/25):

  • Changed from .ply format to my custom .gply format for 3D models and animations
  • Finished basic skeletal animation code, to be polished and re-added when I have an ECS
  • Shader implementation refactor + geometry shader support
  • Experiments not related to engine development (see below)
  • decided to bring back this bloodeh github pages thing

Work in progress (15/06/25):

  • ECS (they're confusing)

About:

This is not a proper game engine yet; it is currently more a framework for 3D graphics because that is what I have spent most time on. I am currently trying to understand sparse-sets, archetypes and other ECS optimistations. Once I have added an ECS interface I am happy with, I can work on integrating a few things I've worked on at the side and adding more game centred features so I can finally make a prototype game. After that I will probably start making the engine an actual engine with an editor, asset management, etc.

The engine development was a little stagnant because I was having more fun experimenting in the engine. These are what I made:
- Recreated the "Final Demo" of Sebastian Lague's software rasterizer of a plane flying through procedural terrain
- 3rd person character controller that I designed in Unity which I then added skeletal animation to (nicked the model from Sebatstain Lague)
- State machine wandering NPC (reminds me of my self ghagahaahaharaahaha) also with skeletal animation
- Realised I was stupid enough to copy code from my UBO tests into a function in my shader implementation without changing any literals to function parameters and not notice for nearly 2 months
